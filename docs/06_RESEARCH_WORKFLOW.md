# 研究流程（Research Workflow）

> 版本：2.0.0 · 冻结日：2026-07-19  
> 服从 `01_CONSTITUTION.md` 与 `PAAF_PROJECT_SPEC.md`。

---

## 1. 单假设原则

每个实验只验证一个主要假设。

禁止同轮同时引入新 Detector、新出场、新过滤器和新仓位。用户明确要求并行 A/B 时，每个实验臂仍须有独立假设与结果。

---

## 2. 标准流程

```text
提出可证伪假设
  ↓
登记实验 ID / 基线 / 指标 / 停止条件
  ↓
接口与单元测试
  ↓
最小实现
  ↓
固定协议回测
  ↓
CSV / 指纹 / 配置落盘
  ↓
分析与不确定性
  ↓
KEEP / REVERT / HOLD
  ↓
证据等级与生命周期更新
  ↓
Commit（仅用户明确要求）
```

---

## 3. 实验登记最低字段

可机读 schema：`experiments/schema.yaml`（新实验必须按该 schema 登记）。

- 实验 ID；
- Detector ID 与版本；
- 当前生命周期 / 证据等级；
- 单句假设；
- 基线 ID；
- 品种与区间；
- 数据协议版本；
- 成本与执行模型；
- 唯一改动变量；
- 主要指标与停止条件；
- 输出 CSV 路径；
- 代码 / 配置指纹；
- 决策（KEEP / REVERT / HOLD）。

---

## 4. 决策

| 决策 | 含义 |
|------|------|
| KEEP | 证据支持继续保留，但不自动进入生产 |
| REVERT | 证据反对假设，回滚或关闭实验臂 |
| HOLD | 样本不足、证据冲突或仅样本内，不晋级 |

KEEP 不是 Production；Production 需要 E4 与用户确认。

---

## 5. 证据晋级

```text
E0 Idea
 ↓ 单品种固定协议 + CSV
E1 Single Symbol
 ↓ 参数冻结 + 多品种
E2 Multi Symbol
 ↓ 多年份或严格 OOS
E3 Multi Year / OOS
 ↓ 执行、成本、日志审计 + 用户确认
E4 Production Ready
```

晋级只允许逐级完成。某一级失败则 HOLD / REVERT，不得因总 PnL 高而跳级。

---

## 6. 废弃流程

单个窗口出现 `PF < 1` 或 `Expectancy < 0`：

1. 若在 Production，立即退出生产 profile 并降到 Testing；
2. 在预登记的独立窗口 / 品种复核；
3. 连续两个独立证据单元为负且样本达标，标 Deprecated；
4. 样本不足则 HOLD；
5. 保留旧版本、指纹、CSV 和弃用原因。

Deprecated 不等于删除历史。

---

## 7. 不追逐利润

禁止以更高收益为实验目标。允许优化的对象只有：

- 研究质量；
- 可解释性；
- 可复现性；
- 统计有效性；
- 生产真实性。

禁止事后挑品种、挑区间、删交易、加过滤器或重选阈值来挽救假设。

---

## 8. Commit 门禁

无 CSV 或等价可审计输出：

- 不得把结果称为已证实；
- 不得更新证据等级；
- 不得为该研究结论请求 Commit。

即使门禁满足，Agent 仍不得擅自 Commit / Push；必须由用户明确授权。
