# PAAF 宪章（长期不变）

> 版本：2.0.0 · 冻结日：2026-07-19  
> 本文件定义项目**哲学与不可谈判原则**。细则见 `02`–`06` 与根目录 `AGENTS.md`。

---

## 一句话目标

**PAAF 不是为了证明 Brooks 正确；PAAF 是为了在中国商品期货上，用接近实盘的数据，发现具有统计显著性的价格行为 Alpha。**

---

## PAAF 是什么

| 是 | 不是 |
|----|------|
| 量化研究平台 | 单一交易策略 |
| 假设 → 证据 → 晋级流水线 | 「感觉对了就上线」 |
| 检测器（Detector）= 可检验假设 | 检测器 = 已证实 Alpha |
| 生产级数据与成本为默认 | 为好看指标换数据/抹成本 |

**PAAF** = Price Action Alpha Framework（价格行为 Alpha 框架）。

每一个 Pattern / OPP 编号，在被统计证据支撑之前，**默认无 Alpha**（零假设）。

---

## 研究哲学

1. **数据 > 理论（Data > Theory）**  
   争议以可复现回测 / CSV / 代码路径为准，不以叙事为准。

2. **证据 > 意见（Evidence > Opinion）**  
   机制推导 ≠ 结论。未证实须标明「待验证假设」。

3. **研究 > 优化（Research > Optimization）**  
   不以回测收益为目标，不因好结果继续扩参。

4. **生产真实性 > 回测好看（Production > Backtest Beauty）**  
   默认使用接近实盘的数据、手续费与滑点；零成本仅作诊断，不得与含成本结论混比。

**在理解之前，禁止优化。**

### 不追逐利润

禁止优化更高回测收益。只优化研究质量、可解释性、可复现性和统计有效性。

---

## 复杂度预算（Complexity Budget）

每一个新模块、新依赖、新算法，必须先回答：

> **它如何提升研究能力（可检验假设、可审计证据、可复现实验）？**

若答不出统计/研究价值 —— **不允许实现**。

典型需先论证再允许的方向（未论证前默认拒绝）：

- Kelly / 贝叶斯仓位
- XGBoost / 深度学习 / Transformer
- 大规模参数搜索、自动优化器
- 与当前假设无关的「顺手重构」

---

## 零假设与晋级

- **零假设**：每个检测器默认无 Alpha。
- **晋级**须统计证据（见 `06_RESEARCH_WORKFLOW.md`、检测器生命周期）。
- 未经 Verified / Production 门禁，不得把假设写成「已证实收益来源」。

---

## 绝对禁止

| 禁止项 | 说明 |
|--------|------|
| 未来函数 / 前视偏差 | 用未来 bar、未收盘信息、复权偷看等 |
| 曲线拟合导向 | 为凑指标改参、缩样本、抹成本 |
| 未经授权的参数搜索 | 网格、连跑优化到目标年化 |
| 隐藏全局状态 | 跨回测泄漏、不可审计的隐式缓存 |
| 机器学习（v0.9 前） | 除非用户书面授权并单独立项 |
| 讨好型结论 | 「终极形态」「方向完全验证」等 |
