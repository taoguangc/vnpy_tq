## 3. 两个可计算标签（核心定义）

### 3.1 标签甲：更大周期结构偏多

**名称（审计字段）**：`htf_bullish_structure`（更大周期看多结构）

**默认可计算规则（提案）** — 三选一必须在确认时钉死为**唯一**条：

| 方案 | 规则（全部用已收盘棒，禁止用未收盘未来棒） |
|------|----------------------------------------------|
| **甲-1（推荐先钉）** | 60 分钟：收盘价 > 60 分钟上的 20 根均线，且该均线斜率 ≥ 0（本棒均线 ≥ 上一根已收盘均线） |
| 甲-2 | 60 分钟：最近 N 根（提案 N=5）高点抬高且低点抬高 |
| 甲-3 | 仅当情境引擎一级状态为「趋势」，且扩展字段趋势偏向为「向上」 |

```text
确认时必须选定甲-1 / 甲-2 / 甲-3 之一。
本草案推荐甲-1：定义短、可复现、不依赖尚未完全产品化的趋势偏向通道。
```

**为真**：满足选定方案。  
**为假**：不满足。  
**未知**：预热不足（均线未满 20 根等）→ 标签记为未知，**过滤实验中视为不允许交易**（不得把未知当成真）。

### 3.2 标签乙：刚站上 / 站稳于均线 20（信号周期）

**名称（审计字段）**：`reclaim_ma20_long`（做多侧收回均线 20）

**默认可计算规则（提案）**：

在信号周期（如 5 分钟）已收盘棒 `i` 上：

1. `ma20[i]` = 收盘价简单移动平均（长度 20），仅用 `i` 及更早已收盘棒；  
2. **站上**：`close[i] > ma20[i]`；  
3. **刚**：上一根已收盘满足 `close[i-1] ≤ ma20[i-1]`（从下方或均线上穿越到上方）；  
4. 可选加强（确认时二选一）：  
   - **乙-松**：只需 2+3（穿越当棒为真）；  
   - **乙-严**：2+3，且下一根已收盘仍 `close[i+1] > ma20[i+1]` 才把「站稳」记在 `i+1`（信号不得用 `i+1` 的未收盘信息；若选乙-严，信号棒改为确认棒）。

```text
草案推荐先钉「乙-松」：先验证标签可审计，再考虑乙-严。
禁止：用当棒未收盘的最高最低去「提前」判定站上。
```

  htf_bullish_structure == 真
