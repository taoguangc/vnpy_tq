# 回测规范（Backtest Spec）

> 版本：2.0.0 · 冻结日：2026-07-19  
> 数据口径见 `07_DATA_SPEC.md`；研究晋级见 `06_RESEARCH_WORKFLOW.md`。

---

## 1. 回测目的

回测用于证伪假设、测量不确定性和验证执行语义，不用于追逐更高收益。

正式回测必须：

- 使用冻结数据基线；
- 使用真实手续费与滑点；
- 固定策略、Detector 与配置版本；
- 产出 CSV 或等价可审计输出；
- 明确区间、品种、样本内 / 样本外属性。

---

## 2. 回测授权与配额

只有用户明确说「跑回测 / 验证 / 对照」才执行。

| 类型 | 上限 | 复盘 | 等确认 |
|------|------|------|--------|
| 探索 / 调参 | 3 次 | 完整四段 | 是 |
| 排障 / 修 bug | 改代码+回测合计 3 次 | 轻量 | 否 |
| 用户授权批量 | 按指令 | 并列汇总 | 完成后一次 |
| `--compare` 对照 | 整命令计 1 次 | 并列汇总 | 视情况 |
| 多品种 `--all` | 脚本 1 次 | 并列汇总 | 完成后一次 |

- 配额用尽立即停止；
- 只改代码不回测不计入；
- 排障轮禁止顺带扩参；
- 禁止拆命令绕过配额；
- 禁止自动优化到目标年化。

---

## 3. 统一验收指标

正式报告必须至少输出：

| 指标 | 含义 |
|------|------|
| Trade Count | 完整回合交易数 |
| Win Rate | 胜率 |
| PF | 盈亏因子 |
| Expectancy | 单笔期望值（净成本） |
| Avg Trade | 平均每笔净盈亏 |
| MAE | 最大不利波动 |
| MFE | 最大有利波动 |
| Sharpe | 夏普比率 |
| Max Drawdown | 最大回撤 |

同时建议输出：总净盈亏、年化收益、成本、平均持有期、分 Detector / setup 贡献。

指标不可用时写 `N/A` 并说明日志或引擎缺口，不得静默省略。没有 MAE/MFE 链路时，不得凭最终 PnL 推断入场质量。

---

## 4. 报告格式

### 完整四段（探索 / 调参）

1. 结果摘要：统一指标 + 是否含成本；
2. 发现问题：可检验陈述；
3. 下一步：默认一个优先方向；
4. 等待确认。

### 轻量（排障）

指标变化 + 通过 / 未通过 + 一条原因。

### 并列汇总（对照 / 多品种）

一张并列表 + 一句总结论。

---

## 5. 对照纪律

- 基线与实验臂使用同一数据、区间、成本、资金与执行模型；
- 只允许一个主要变量；
- 参数确定后再跑 OOS，OOS 不重挑阈值；
- 不只报告盈利品种；
- 零成本仅作诊断，不能与含成本结论混比；
- 包路径与生成的单文件路径应数值对齐。

---

## 6. 执行真实性

- 信号在已收盘的合成 K 上计算；
- 风控在最小可执行周期（默认 1m）执行；
- 同 bar 同时触发止损与止盈时止损优先；
- 跳空按可成交边界处理，不用收盘价伪造理想成交；
- 换月成本须按数据层、引擎层、触发层分别验证。

---

## 7. Windows PowerShell

```powershell
Set-Location C:\projects\vnpy_tq; $env:PYTHONIOENCODING='utf-8'; .\.venv\Scripts\python.exe ...
```

多语句用 `;`。乱码临时文件 `_tmp_*` / `_tem_*` 必须当轮删除。

---

## 8. 结论边界

- 盈利不等于 Alpha；
- 单品种正收益最多支持 E1；
- 多品种结果必须报告异质性；
- 样本内通过不等于 E3；
- 没有成本、CSV 或版本指纹，不得晋级。
